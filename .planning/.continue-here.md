<decisions_made>

- Center alignment is the v1 default for individual top-level prompt content.
- `prompt_align` is fundamental enough to be a current optional v1 extension rather than deferred future work.
- `prompt_align` is sticky until the next `prompt_align` or `prompt_begin`.
- `prompt_align` affects top-level items only; children inside `row` and `button_group` keep their own cell layout convention.
- Unknown or empty `prompt_align` is ignored and preserves current alignment.
- `prompt_size` still differs: invalid size clears the pending size hint instead of preserving an older value.
- `prompt_input` is not v1. It remains reserved future work because lifecycle, escaping, validation, frontend variable writes, multi-frontend races, console visibility, and fallback semantics are unresolved.
</decisions_made>
